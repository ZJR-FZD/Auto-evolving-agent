● 这不是 bug，是正常的超时错误。Playwright
  在访问外部网站时超时了（30秒内没加载完），browser-service 正确地返回了 500
  错误，然后 harness 的 browser_navigate 工具会把它包装成 {"ok": false, "error":
   "navigate failed: ..."} 返回给 agent。

  从 trajectory 里也能看到 agent 收到这类错误后会换策略（比如改用
  search_text），这是预期行为。

  不过有一点可以优化：当前 browser_navigate 的超时异常没有被 route handler
  捕获，直接抛到了 ASGI 层变成 500。更好的做法是在 route handler 里 catch
  TimeoutError 并返回一个结构化的 JSON 错误响应（比如 {"ok": false, "error": 
  "timeout"}），而不是让 FastAPI 返回裸 500。这样日志会更干净，也避免了不必要的
  traceback 刷屏。
